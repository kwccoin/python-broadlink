Very possibly the same issue as #535 but occurs with just one device. I'm using a Broadlink RM Mini 3 D.

Log looks like:

[1/15/2020, 12:50:25 AM] [Broadlink RM] [INFO] Learn Code initializing (undefined)
[1/15/2020, 12:50:25 AM] [Broadlink RM] Learn Code (ready)
[1/15/2020, 12:50:35 AM] [Broadlink RM] [INFO] Learn Code (stopped - 10s timeout)
[1/15/2020, 12:50:35 AM] [Broadlink RM] [INFO] Learn Code (stopped)

Tried running homebridge on both raspbian and macOS both with the same result.

Devices gets discovered with no problem:

[Broadlink RM] [INFO] Discovered Broadlink RM Mini 3 D (5f36) at 192.168.1.8 (a7:df:24:xx:xx:xx)

I've been debugging this issue but I'm stuck in the message callback inside the broadlinkjs-rm module. There, the bytes at 0x22 and 0x23 are always 251 and 255, which causes the callback to return. The corresponding code is:

const err = response[0x22] | (response[0x23] << 8);
if (err != 0) {
  return;
}

I don't know about the protocol that Broadlink implements, so I don't know what those error codes might mean.
